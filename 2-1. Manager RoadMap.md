# 🚀 프로젝트 로드맵
**EDAS Control Manager**
여러 개의 독립적인 EDAS 진단 실행 파일(.exe)을 중앙에서 관리하고 제어하기 위한 통합 관리 시스템 개발 
각 App의 실행 상태를 실시간으로 모니터링하며, MSMQ(Microsoft Message Queue)를 통해 양방향 통신 제어를 수행

---

## ## 1. 시스템 개요
본 시스템은 **C# .NET Framework 4.8** 환경에서 동작하며, 다수의 프로세스를 효율적으로 제어하기 위해 **인터페이스 기반 설계**와 **비동기 메시징** 방식을 채택하고 있습니다.

| 구분 | 상세 내용 |
| :--- | :--- |
| **관리 대상** | EDAS 진단 EXE 애플리케이션 |
| **통신 방식** | `System.Messaging.MessageQueue` (MSMQ) 기반 |
| **UI 구조** | WinForm (ToolStripMenuItem + ListView) |
| **주요 기능** | 등록, 실행(Execute), 정지(Stop), 화면 제어(Show/Hide), 상태 모니터링 |

---

## ## 2. 핵심 아키텍처 및 통신 모델

### ### 메시지 프로토콜 (`MessageInfo` 클래스)
관리자와 개별 App 간의 데이터 교환을 위해 정의된 표준 규격입니다.

* **MessageType**: `ReqProcessStatus`, `ReqRunningStart`, `ReqRunningStop`, `ReqProcessShow`, `ReqProcessHide` 등 요청과 응답을 구분하는 열거형.
* **Property**: `Sender`, `Receiver`, `MsgType`, `MsgBody`.
* **MsgBody 상태값**: `Execute`, `Closing`, `Closed`, `Run`, `Stop`, `Timeout`.

### ### 실시간 상태 동기화 (Tick 기반)
Timer를 통해 주기적으로 모든 등록된 App에 `ReqProcessStatus`를 송신합니다. 각 App으로부터 수신된 `ResProcessStatus` 메시지를 분석하여 ListView의 **Status 컬럼**을 실시간으로 업데이트합니다.

---

## ## 3. 주요 기능 및 제어 로직

### ### 📂 애플리케이션 관리
* **중복 방지**: 동일한 이름의 App이 등록될 경우 알람 창을 통해 중복 등록을 차단합니다.
* **동적 등록**: 파일 탐색기를 통해 EXE 경로를 지정하면 ListView에 즉시 추가되며 초기 상태는 `Closed`로 설정됩니다.
* **안전 삭제**: App 상태가 `Closed`일 때만 목록에서 삭제가 가능하여 오작동을 방지합니다.

### ### 🕹️ 제어 방식
1.  **전체 제어 (Control All)**: 상단 메뉴를 통해 모든 App을 순차적으로 실행하거나 종료합니다. (일괄 실행 시 중복 실행 알람은 생략하여 사용성 향상)
2.  **개별 제어 (Context Menu)**: ListView의 각 행을 우클릭하여 특정 App에 대한 정밀 제어 명령을 하달합니다.

---

## ## 4. 테스트 전략

### ### 단위 테스트 (MSTest)
* **TestHooks 도입**: private 로직이나 통신 상태를 테스트하기 위한 Hook을 구성하여 `MSTest` 프레임워크 내에서 유닛 테스트를 수행합니다.
* **검증 항목**: App 등록/삭제 로직, 메시지 타입별 처리 결과, 중복 실행 방지 조건 등.
