# 🚀 프로젝트 로드맵
**EDAS Data Relay Service**
클라이언트와 외부 서버/DB 사이에서 데이터를 중계하는 WCF 기반 통합 데이터 레이어 구축
파일 시스템 접근 및 MSSQL 쿼리 실행을 통해 클라이언트가 필요한 정보를 통합적으로 제공

---

## ## 1. 시스템 개요
본 시스템은 **WCF(Windows Communication Foundation)**를 기반으로 하며, IIS에 호스팅되어 다종의 프로토콜을 통해 서버 간 데이터 통신을 중계합니다.

| 구분 | 상세 내용 |
| :--- | :--- |
| **호스팅 환경** | IIS (Internet Information Services) "웹사이트 추가" |
| **지원 프로토콜** | HTTP (80 포트), net.tcp (8080 포트) |
| **데이터 모델** | `Communication.Data.ExecuteInfo` (기존 라이브러리 유지) |
| **핵심 역할** | 특정 파일 데이터 추출 및 MSSQL DB 조회 중계 |

---

## ## 2. 통신 아키텍처 및 프로토콜

### ### 서비스 계약 (Service Contract)
모든 데이터 교환은 `ExecuteInfo` 객체를 단일 창구로 사용하며, 내부 `ProcedureName`에 따라 로직을 분기합니다.

* **인터페이스**: `IDataRelayService`
* **메서드**: `ExecuteResponse GetData(ExecuteInfo request)`
* **데이터 흐름**: Client → WCF Relay → File Server OR MSSQL DB → Client

### ### IIS 및 바인딩 설정
* **HTTP Binding**: 방화벽 유연성이 필요한 외부 연결용.
* **Net.Tcp Binding**: 내부망 내 고성능/고효율 이진 통신용.
* **추가 설정**: IIS의 '사이트 바인딩'에서 80(http)과 8080(net.tcp)을 설정하고, WAS(Windows Process Activation Service) 기능을 통해 비-HTTP 프로토콜을 활성화합니다.

---

## ## 3. 주요 비즈니스 로직 (ExecuteInfo 처리)

### ### 📂 Case 1: 파일 시스템 기반 정보 추출 (`GetSignalInfo`)
`ProcedureName`이 **"GetSignalInfo"**인 경우, 전달된 파라미터를 파싱하여 외부 파일 서버의 데이터를 읽어옵니다.

1. **파라미터 매핑**:
   * `FileName`, `EquipmentID`, `ValueType` → **String 변수 할당**
   * `SamplingCount` → **Int32 변수 할당**
2. **데이터 처리**:
   * 지정된 경로의 파일을 `ReadLine`으로 순차적으로 읽어 필요한 정보를 추출합니다.
   * 추출된 결과값을 `ExecuteInfo` 응답 객체에 담아 반환합니다.

### ### 🗄️ Case 2: MSSQL 데이터베이스 조회
`ProcedureName`이 **"GetSignalInfo"가 아닌 모든 경우**에 해당합니다.

* **동작 방식**: `ExecuteInfo.Procedure` 정보를 바탕으로 MSSQL 쿼리를 생성하거나 저장 프로시저를 호출합니다.
* **범용성**: `Communication.Data` 구조를 그대로 유지하므로, 다양한 DB 조회 요청을 하나의 서비스 엔드포인트에서 처리 가능합니다.

---

## ## 4. 배포 전략

* **App.config/Web.config**: `serviceMetadata`를 활성화하여 클라이언트에서 서비스 참조를 추가할 수 있도록 설정합니다.
* **권한 설정**: WCF 서비스가 구동되는 IIS 앱 풀 계정이 외부 서버의 파일 경로 및 MSSQL DB에 접근할 수 있는 권한을 부여해야 합니다.
