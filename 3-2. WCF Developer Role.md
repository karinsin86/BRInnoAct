# 🚀 개발 역할 분담

## 1. 개발자별 역할 분담 (3인 구성)

| 담당자 | 핵심 역할 (Role) | 주요 담당 컴포넌트 (`Class/Service`) | 분담 사유 및 기대 효과 |
| :--- | :--- | :--- | :--- |
| **장봉익 프로(역할1)** | **서비스/계약·호스트** | `DataService.cs`, `Web.config` 등 바인딩/엔드포인트, 공통 입력 검증 | 클라이언트가 붙는 **API 표면**  |
| **최홍열 프로(역할2)** | **데이터 액세스/공통 DTO 실행 경로** | `Communication.Data`, `ConnectionInfo`, `ExecuteInfo`, `Procedure` 등 DB 연결/타임아웃/에러 매핑 등 | **저장 프로시저 실행** |
| **함종우 프로(역할3)** | **비즈니스/인프라 핸들러** | `GetSignalInfoHandler`, `GetDataBaseHandler`, `appSettings` 등 | **Execute 분기**, **구체적인  동작** |

---

## 2. 단계별 진행 계획

프로젝트의 안정적인 빌드와 병렬 개발을 위해 총 4단계로 진행합니다.

### **Step 0:합의**
- `IDataRelayService`와 `DataContract` 필드·네임스페이스 확정.  
- `ExecuteInfo` → 성공/실패 시 `ExecuteResult` 규칙(널 허용 여부 등) 정리.  
- Git 브랜치: `main` + 기능 브랜치, 또는 A가 계약만 먼저 PR로 고정.

### **Step 1: 병렬 스켈레톤**
- **역할 1**: `Execute` 시그니처, null 시 `FaultException`, IIS/바인딩까지 로컬에서 호출
- **역할 2**: `DataRequest` + 테스트용 연결 문자열로 한 프로시저 경로 구축
- **역할 3**: `GetSignalInfo` 테스트 파일 경로로 최소 구현, 설정 키 이름만 합의

### **Step 2: 연결**
- **역할 1**: `DataService`에서 B/C 핸들러만 호출하도록 **얇은 디스패치** 유지.  
- **통합**: WCF Test Client 또는 단위 테스트로 `Execute` 두 갈래 검증.

### **Step 3: 기능 본구현**
- **역할 3**: 기능 서비스의 TODO(실제 전달 대상, 재시도, 타임아웃).  
- **역할 1**: 메시지 크기·바인딩과 맞는지 `Web.config` 조정.

### **Step 4: 품질/운영**
- 테스트 프로젝트 기준으로 시나리오 보강.  
- 로깅, 보안 모드(net.tcp `security mode` 등) 검토.
